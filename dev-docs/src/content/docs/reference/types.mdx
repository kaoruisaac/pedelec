All types below are exported from `@kaoruisaac/pedelec`.

```ts
import type {
  PedelecOptions,
  ProviderCode,
  ProviderInfo,
  PedelecSessionStatus,
  ToolArgsSchema,
  SandboxAsset,
  SandboxAssetPath,
} from "@kaoruisaac/pedelec";
```

## Client and session input

### `PedelecOptions`

```ts
type PedelecOptions = {
  bridgeTimeoutMs?: number;
};
```

### `ProviderCode`

```ts
type ProviderCode =
  | "codex"
  | "gemini"
  | "opencode"
  | "cursor"
  | "claude"
  | "ollama";
```

### `CreateSessionInput`

```ts
type CreateSessionInput<
  TTools extends readonly ToolDefinition[] = readonly ToolDefinition[],
> =
  | {
      provider: ProviderCode;
      model?: string;
      skills?: SkillsInput<TTools>;
      autoEndOnDisconnect?: boolean;
    }
  | {
      provider?: undefined;
      model?: never;
      skills?: SkillsInput<TTools>;
      autoEndOnDisconnect?: boolean;
    };
```

The union prevents `model` without `provider` in TypeScript.

## Provider and settings

### `ProviderInfo`

```ts
type ProviderInfo = {
  name: string;
  code: ProviderCode;
  path: string | null;
  available: boolean;
  error: string | null;
};
```

### `PedelecSettings`

```ts
type PedelecSettings = {
  defaultProvider: ProviderCode | null;
  defaultModels: Partial<Record<ProviderCode, string>>;
};
```

### `ApprovalStatus`

```ts
type ApprovalStatus = {
  installed: boolean;
  approved: boolean;
  origin: string | null;
};
```

### `PedelecAvailability`

```ts
type PedelecAvailability = {
  available: boolean;
  extension: { available: boolean };
  approval: { approved: boolean; origin: string | null };
  desktop: { available: boolean; launchAttempted: boolean };
  error?: PedelecError;
};
```

`available` requires every layer. `launchAttempted` only means the SDK sent the `getSettings()` Desktop probe; it does not confirm a process launch. A settings protocol error is reported as Desktop unavailable.

## Errors and status

## Assets

```ts
type SandboxAssetPath = `input/${string}`;

type SandboxAsset = {
  name: string;
  path: SandboxAssetPath;
  sizeBytes: number;
  modifiedAt: number;
};
```

`modifiedAt` is the sandbox filesystem modification time. `path` is sandbox-relative and does not reveal the browser user's absolute local path.

### `PedelecError`

```ts
type PedelecError = {
  code: string;
  message: string;
  details?: unknown;
};
```

Do not assume `details` has one global shape. Narrow by `code` and validate before reading.

### `PedelecSessionStatus`

```ts
type PedelecSessionStatus =
  | "idle"
  | "running"
  | "waiting_tool_result"
  | "ended"
  | "error";
```

## JSON value types

### `JsonPrimitive`

```ts
type JsonPrimitive = string | number | boolean | null;
```

### `JsonValue`

```ts
type JsonValue =
  | JsonPrimitive
  | JsonValue[]
  | { [key: string]: JsonValue };
```

Used by tool schema metadata. Runtime tool results should follow the same practical JSON-compatible constraints even when their TypeScript return type is broader.

## Tool definitions

### `ToolSpecificHandler`

```ts
type ToolSpecificHandler<TArgs = unknown, TResult = unknown> = (
  args: TArgs,
  ctx: ToolCallContext,
) => TResult | Promise<TResult>;
```

### `ToolDefinition`

```ts
type ToolDefinition<
  TArgs = unknown,
  TResult = unknown,
  TName extends string = string,
> = {
  name: TName;
  description: string;
  argsSchema: ToolArgsSchema;
  timeoutMs?: number;
  handler?: ToolSpecificHandler<TArgs, TResult>;
};
```

### `SkillsInput`

```ts
type SkillsInput<
  TTools extends readonly ToolDefinition[] = readonly ToolDefinition[],
> = {
  guidance: string;
  tools: TTools;
};
```

### `ToolNameOf`

```ts
type ToolNameOf<TTools extends readonly ToolDefinition[]> = Extract<
  TTools[number]["name"],
  string
>;
```

### Serializable manifests

```ts
type SerializableToolManifest = {
  name: string;
  description: string;
  argsSchema: ToolArgsSchema;
  timeoutMs?: number;
};

type SerializableSkillsManifest = {
  guidance: string;
  tools: SerializableToolManifest[];
};
```

Inline handlers are not part of the serialized manifest.

## Tool schema metadata

```ts
type ToolArgsSchemaMeta<
  TDefault extends JsonValue = JsonValue,
> = {
  description?: string;
  default?: TDefault;
  examples?: TDefault[];
};
```

## Tool schema nodes

### String

```ts
type ToolArgsStringSchema = ToolArgsSchemaMeta<string> & {
  type: "string";
  enum?: string[];
  minLength?: number;
  maxLength?: number;
  pattern?: string;
};
```

### Number

```ts
type ToolArgsNumberSchema = ToolArgsSchemaMeta<number> & {
  type: "number";
  enum?: number[];
  minimum?: number;
  maximum?: number;
};
```

### Integer

```ts
type ToolArgsIntegerSchema = ToolArgsSchemaMeta<number> & {
  type: "integer";
  enum?: number[];
  minimum?: number;
  maximum?: number;
};
```

### Boolean

```ts
type ToolArgsBooleanSchema = ToolArgsSchemaMeta<boolean> & {
  type: "boolean";
  enum?: boolean[];
};
```

### Array

```ts
type ToolArgsArraySchema = ToolArgsSchemaMeta<JsonValue[]> & {
  type: "array";
  items: ToolArgsSchemaNode;
  minItems?: number;
  maxItems?: number;
  uniqueItems?: boolean;
};
```

### Object

```ts
type ToolArgsObjectSchema = ToolArgsSchemaMeta<
  Record<string, JsonValue>
> & {
  type: "object";
  properties?: Record<string, ToolArgsSchemaNode>;
  required?: string[];
};
```

### One-of

```ts
type ToolArgsOneOfSchema = ToolArgsSchemaMeta & {
  oneOf: ToolArgsSchemaNode[];
};
```

### Node and root aliases

```ts
type ToolArgsSchemaNode =
  | ToolArgsStringSchema
  | ToolArgsNumberSchema
  | ToolArgsIntegerSchema
  | ToolArgsBooleanSchema
  | ToolArgsArraySchema
  | ToolArgsObjectSchema
  | ToolArgsOneOfSchema;

type ToolArgsSchema = ToolArgsObjectSchema;
```

The root alias enforces an object schema at compile time.

## Event contexts

### `PedelecEventContext`

```ts
type PedelecEventContext = {
  sessionId: string;
  provider: string;
  model?: string;
  sessionCreatedAt: number;
  eventReceivedAt?: number;
  eventEmittedAt: number;
  turnId?: string;
  turnStartedAt?: number;
  turnKind?: "user" | "prepare";
  source: "core" | "sdk";
};
```

### `ChatEventContext`

```ts
type ChatEventContext = PedelecEventContext & {
  type: "chat_delta";
  turnId: string;
  turnStartedAt: number;
  eventReceivedAt: number;
};
```

### `ToolCallContext`

```ts
type ToolCallContext = PedelecEventContext & {
  type: "tool_call";
  toolRequestId: string;
  tool: string;
  turnId: string;
  turnStartedAt: number;
  eventReceivedAt: number;
};
```

### `StatusEventContext`

```ts
type StatusEventContext = PedelecEventContext & {
  type: "status_changed" | "sdk_status_changed";
  status: PedelecSessionStatus;
  previousStatus: PedelecSessionStatus;
};
```

### `ErrorEventContext`

```ts
type ErrorEventContext = PedelecEventContext & {
  type: "error" | "sdk_error";
};
```

### `EndedEventContext`

```ts
type EndedEventContext = PedelecEventContext & {
  type: "ended" | "sdk_ended";
};
```

Fields marked optional in the base context are present only when relevant. Treat IDs as opaque and timestamps as browser diagnostics, not security claims.
